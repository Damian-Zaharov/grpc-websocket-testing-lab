Professional AQA project demonstrating expertise in testing distributed systems using asynchronous protocols: **gRPC** (control plane) and **WebSockets** (data plane).

The project simulates a **"Ticker Service"** microservice:
1. **Control Plane (gRPC):** Handles subscriptions/unsubscriptions for specific tickers (e.g., `BTC/USD`).
2. **Data Plane (WebSocket):** Streams real-time price updates.

## Framework Architecture
The framework follows a **Layered Architecture** approach:

- **Clients Layer:** Async adapters for gRPC and WS protocols that encapsulate connection logic.
- **Service Layer (Helpers):** High-level business logic (e.g., "subscribe and wait for a specific message"). Features smart message filtering to ensure test isolation.
- **Data Layer (Pydantic):** Strict validation of JSON schemas and business rules (price ranges, ticker formats).
- **Report Layer (Allure):** Full integration with steps, attachments, and metadata (epics, features, severity).

## Test Coverage
The suite includes 9 automated tests covering key reliability aspects:
*   🔵 **E2E Flow:** Full cycle from gRPC subscription to receiving live data.
*   🔵 **Contract Testing:** Validation of gRPC service definitions and server responses.
*   🔵 **Resilience:** Testing the system's ability to recover streaming after a connection drop (Reconnect).
*   🔵 **Business Logic:** Verification of the price change mathematical model (Delta check).
*   🔵 **Idempotency:** Ensuring no duplicate traffic is generated by multiple identical subscriptions.
*   🔵 **Load Lite:** Mass subscription for 20+ concurrent tickers.
*   🔵 **Negative Testing:** Handling empty values and malformed data.

## Quick Start (CI/CD Ready)
The project is fully containerized. No local Python installation is required:

```bash
# Build and run all tests in an isolated Docker network
docker-compose up --build --abort-on-container-exit
```

### Reporting
Test results are available in the `allure-results` folder after execution:
```bash
allure serve allure-results
```

## Tech Stack
- **Backend:** Python 3.11, `grpcio`, `websockets`.
- **Testing:** `pytest`, `pytest-asyncio`, `allure-pytest`.
- **Validation:** `pydantic v2`.
- **Infrastructure:** `Docker`, `Docker Compose`, `GitHub Actions`.

# gRPC & WebSocket Testing Lab
Профессиональный учебный проект AQA, демонстрирующий навыки автоматизации тестирования распределенных систем с использованием асинхронных протоколов **gRPC** и **WebSockets**.

## Описание решения
Проект имитирует микросервис «Биржевой монитор» (Ticker Service):
1. **Control Plane (gRPC):** Принимает команды на подписку/отписку от тикеров (например, `BTC/USD`).
2. **Data Plane (WebSocket):** Транслирует потоковые данные об изменении цен в реальном времени.

## Архитектура тестового фреймворка
Фреймворк построен с использованием многоуровневого подхода (Layered Architecture):

- **Clients Layer:** Асинхронные адаптеры для gRPC и WS, инкапсулирующие логику подключения и протоколов.
- **Service Layer (Helpers):** Высокоуровневая бизнес-логика тестов (например, "подписка + ожидание конкретного сообщения в стриме"). Реализована умная фильтрация сообщений для изоляции состояния тестов.
- **Data Layer (Pydantic):** Строгая валидация JSON-схем и бизнес-правил (например, проверка диапазонов цен и форматов тикеров).
- **Report Layer (Allure):** Интеграция с шагами (`steps`), вложениями (`attachments`) и метаданными (`epics`, `features`).

## Покрытые сценарии
Проект включает в себя 9 автоматизированных тестов, покрывающих ключевые аспекты надежности:
*    🔵 **E2E Flow:** Полный цикл от подписки до получения котировок.
*    🔵 **Contract Testing:** Валидация gRPC контрактов и ответов сервера.
*    🔵 **Resilience:** Проверка автоматического восстановления стриминга после разрыва соединения (Reconnect).
*    🔵 **Business Logic:** Проверка математической модели изменения цены (Delta check).
*    🔵 **Idempotency:** Тестирование защиты от дублирования трафика.
*    🔵 **Load Testing:** Массовая подписка на 20+ тикеров одновременно.
*    🔵 **Negative Testing:** Обработка пустых значений и некорректных форматов данных.
